Detailed Description

File containing OpenSSL and Standard includes, as well as the declarations of all HMAC-Tag generation functions.

Author
Eduardo Andrade
Date
Mar 2020 This header file declares the several funtions implemented on the file hmac_functions.c. Also, this file includes all of the other files and libraries used on the HMAC context (OpenSSL, C Standard Libraries).
Note
This library was tested using OpenSSL 1.1.1 library.
See also
https://www.openssl.org/docs/man1.1.1/man3/HMAC.html
https://www.openssl.org/docs/man1.1.1/man7/evp.html

)

Function that generates an HMAC-BLAKE2b_80 Tag.

This function generates an HMAC Tag of 80bits (10bytes) long, using BLAKE2b as its base hashing algorithm. It receives data, key and dest as pointers, and both data and key sizes as size_t. The functions calculates the MAC tag and stores it on dest. It uses OpenSSL Library to implement such algorithms.

Below is and example of usage:

uint8_t* dest = NULL;
uint8_t* key = (uint8_t*)malloc(sizeof(uint8_t)*20);
set_key(key); // pseudo-function that populates key
uint8_t* data = (uint8_t*)malloc(sizeof(uint8_t)*8);
set_data(data); // pseudo-function that populates data
int data_size = 8, key_size = 20;
hmac_BLAKE2b_80(data, key, data_size, key_size, &dest);
print_array_hex(dest); // pseudo-function that prints dest in hex format
Parameters
dataPointer (uint8_t*) containg the data that will be used to generate the HMAC Tag
keyPointer (uint8_t*) containg the key that will be used to generate the HMAC Tag
data_sizeVariable (size_t) that hold the size in bytes of data.
key_sizeVariable (size_t) that hold the size in bytes of key.
destkey Pointer (uint8_t**) pointing to the destiny memory address where HMAC tag should be stored
Returns
The function doesn't return any value
Warning
dest should be create as a data type capable of storing the HMAC Tag (ex. uint8_t*). However, it's memory address must be passed to the function and not the pointer itself (&dest)
