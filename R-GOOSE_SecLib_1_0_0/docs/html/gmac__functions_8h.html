Detailed Description

File containing OpenSSL and Standard includes, as well as the declarations of all GMAC-Tag generation functions.

Author
Eduardo Andrade
Date
Mar 2020 This header file declares the several funtions implemented on the file gmac_functions.c. Also, this file includes all of the other files and libraries used on the GMAC context (OpenSSL, C Standard Libraries).
Note
This library was tested using OpenSSL 1.1.1 library.
See also
https://www.openssl.org/docs/man1.1.1/

Function that generates an gmac_AES128_128 Tag.

This function generates an GMAC Tag of 128bits (16bytes) long, using AES128 as its base cryptographic algorithm. It receives data, key, iv and dest as pointers, and both data and IV sizes as size_t. The functions calculates the MAC tag and stores it on dest. It uses OpenSSL Library to implement such algorithms. To provide only authentication and data integrity, data is passed to OpenSSL as AAD(Additional Authenticated Data), leaving the PT(Plain Text) empty.

Below is and example of usage:

uint8_t* dest = NULL;
uint8_t* key = (uint8_t*)malloc(sizeof(uint8_t)*16);
set_key(key); // pseudo-function that populates key
uint8_t* data = (uint8_t*)malloc(sizeof(uint8_t)*16);
set_data(data); // pseudo-function that populates data
uint8_t* iv = (uint8_t*)malloc(sizeof(uint8_t)*12);
set_iv(iv); // pseudo-function that populates IV
int data_size = 16, iv_size = 12;
gmac_AES128_128(data, key, iv, data_size, iv_size, &dest);
print_array_hex(dest); // pseudo-function that prints dest in hex format
Parameters
dataPointer (uint8_t*) containg the data that will be used to generate the GMAC Tag
keyPointer (uint8_t*) containg the key that will be used to generate the GMAC Tag
ivPointer (uint8_t*) containg the initialization vector that will be used to generate the GMAC Tag
data_sizeVariable (size_t) that hold the size in bytes of data.
iv_sizeVariable (size_t) that hold the size in bytes of iv.
destkey Pointer (uint8_t**) pointing to the destiny memory address where GMAC tag should be stored
Returns
The function returns 0 if everything went as expected (no errors) or 1 if an error occured.
Warning
dest should be create as a data type capable of storing the GMAC Tag (ex. uint8_t*). However, it's memory address must be passed to the function and not the pointer itself (&dest)
Note
It is not required to manually allocate/reserve memory for dest, this functions allocates the necessary memory to store the GMAC tag.
