Detailed Description

File containing OpenSSL and Standard includes, as well as the declarations of all AES Algorithm functions.

Author
Eduardo Andrade
Date
Mar 2020 This header file declares the several funtions implemented on the file aes_crypto.c. Also, this file includes all of the other files and libraries used on the AES context (OpenSSL, C Standard Libraries).
Note
This library was tested using OpenSSL 1.1.1 library.
See also
https://www.openssl.org/docs/man1.1.1/man7/evp.html

Function that decrypts feeded data using AES-GCM with 128 bits long key.

This functions decrypts feeded data using AES-GCM-128, producing an output with the same length as the encrypted (original). It receives data, key and dest as pointers, and iv, data_size and iv_size as integers. The function calculates the decrypted data and stores it on dest. It uses OpenSSL Library to implement such algorithm.

Below is an example of usage:

uint8_t* dest = NULL;
uint8_t* key = (uint8_t*)malloc(sizeof(uint8_t)*16);
set_key(key); // pseudo-function that populates key
uint8_t* data = (uint8_t*)malloc(sizeof(uint8_t)*8);
set_data(data); // pseudo-function that populates data
uint8_t* iv = (uint8_t*)malloc(sizeof(uint8_t)*20);
set_data(iv); // pseudo-function that populates iv
int data_size = 8, key_size = 16, iv_size = 20;
int len = aes_128_gcm_decrypt(data, key, iv, data_size, iv_size, &dest);
print_array_hex(dest,len); // pseudo-function that prints dest in hex format
Parameters
dataPointer (uint8_t*) containg the data that will be decrypted
keyPointer (uint8_t*) containg the key that will be used decrypt data
ivPointer (uint8_t*) containg the Initialization Vector to be used by AES
data_sizeVariable (size_t) that hold the size in bytes of data.
iv_sizeVariable (size_t) that hold the size in bytes of IV.
destkey Pointer (uint8_t**) pointing to the destiny memory address where decrypted data should be stored
Returns
An integer containing the length of the decrypted data
Warning
dest should be create as a data type capable of storing the decrypted data (ex. uint8_t*). However, it's memory address must be passed to the function and not the pointer itself (&dest)
Note
It is not required to manually allocate/reserve memory for dest, this functions allocates the necessary memory to store the decrypted data.
